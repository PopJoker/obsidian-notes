# GusApp ForInstall 專案開發與安裝重點筆記

此專案是一個基於 **Flutter** 開發的跨平台行動應用程式，主要用於能源管理系統（EMS）或電池系統的安裝、調試與現場監控。

## 1. 核心技術棧
- **開發框架**: Flutter (Dart)
- **狀態管理**: Provider 或 GetX (視具體版本而定)
- **通訊協議**: 支援透過 HTTP/HTTPS 與後端 API 通訊，並包含 WebSocket 用於即時數據流。
- **介面設計**: 遵循 Material Design，針對手持設備（手機/平板）進行優化，便於現場安裝工程師使用。

## 2. 專案主要功能
- **設備掃描與初始化**: 用於現場安裝時對設備進行標籤掃描或網絡配置。
- **即時監控數據**: 顯示電池組（Pack）或單體電池（Cell）的電壓、電流、溫度等關鍵指標。
- **安裝紀錄管理**: 紀錄安裝過程中的各項參數，並上傳至伺服器備份。

## 3. 目錄架構重點
- `lib/`: 核心代碼。
    - `/views` 或 `/pages`: 安裝引導、監控面板、設置頁面。
    - `/models`: 定義電池包（Pack）與系統狀態的數據模型。
    - `/services`: 處理網絡請求與資料庫交互邏輯。
    - `/widgets`: 自定義的圖形化組件（如電量表、溫度趨勢圖）。
- `assets/`: 存放專案所需的靜態圖標、配置檔或預設樣式。

## 4. 關鍵開發邏輯
- **UI 狀態持久化**: 針對 FAE 在現場操作的需求，介面元件的擺放位置與偏好設定需在應用重啟後保持一致。
- **錯誤處理機制**: 針對現場可能出現的網絡不穩，需實作強健的斷線重連與數據緩存機制。
- **跨平台適配**: 同時支持 Android 與 iOS，需注意不同系統對網絡權限（尤其是非加密 HTTP）的限制。
